перестановки.
задача заключается в перестановки множества из n элементов. Рассмотрим множество
M = [1, ..., n]
для перестановок можно использовать метод уменьшения размера задачи.
для генерации перестановок n элементов можно воспользоваться n-1! перестановкой множества [1,...,n-1]
тогда нам потребуется поставить в n позиций символ n в итоге получим n*n-1! = n! комбинаций
в таком случае мы можем расставлять элемент n начиная слева или справа. Оказывается что лучше
начинать справа и менять направления на каждой следующей итерации. такой подход удовлетворит
требованию минимальным изменений. 1 перестановка за одну итерацию.
Алгоритм Джонсона‐Троттера.
подходит для генерации перестановок при небольших n.
начнём с генерации начальной комбинации и каждого символа будет стрелочка указывающая влево.
введём понятие мобильная компонента та компонента которая указывает на младшую компоненту
алгоритм:
    1.берём максимальную мобильную компоненту
    2.меняем местами её с компонентой на которую она указывает
    3.меняем все стрелки компонент больших максимальной мобильной компоненты на этой итерации

заметим что в таком виде алгоритм применим только для отсортированных по возрастанию множеств.
